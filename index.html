Inspired from the game Portal, from Valve.

Breach is an attempt to recreate the core experience of the game Portal from Valve.

The experience uses some OpenGL techniques to achieve the visual experience of a spacial breach in the world (a teleportation tunnel toward another place).

Hey, what is this stuff?

Breach will become a 3D spacial puzzle game.
It is inspired by the game Portal from Valve.
But you've guessed it, this game has nothing to do with its famous big brother: we're talking about breaches, not portals!

It will leverage multiple kinds of breaches:

  • Uni- or bi-directionnal
  • Static or user controlled
  • Small for objects or big enough for the player
  • Filtering by kind of objects
  • Two breaches only or many interlinked breaches
  • Grouped breaches, linking toward a random destination

I'm a geek, what is it made of?

Hello geeky, I knew you would come here! Everything's fine 'till now?

Breach is a Linux OpenGL software under new BSD licence.
It's entirely written in C++ and uses a single makefile to build, test, run, and generate documentation.

You may want to take a look at the INSTALL.textile file in the project root.

Requirements

  • A computer with a graphic card, even a dummy one
  • A graphical window manager. Sorry console addicts, this game is not CLI and do not use ncurse or libcaca for rendering...
  • Libraries development headers like freeglut3-dev, libpng12-dev, libsigc++-2.0-dev (maybe a bunch of other as well, but I do not depend on them directly)
  • Compilation tools like those given by build-essential
  • Some programming and compilation skills and time, if compilation is problematic

Compiling and running

Simply issue the following lines: 

cd Breach
make compile
make run

As compilation is needed for running the game, you may simply issue the last command.

If you wish to generate the documentation, issue the following: 

make doc

You will find the documentation under Breach/doc/html/index.html.
If you have a LATEX distribution, you can find the PDF under Breach/doc/latex/refman.pdf.
If you don't, the makefile may prompt some harmless errors or warnings, you can safely ignore them.
